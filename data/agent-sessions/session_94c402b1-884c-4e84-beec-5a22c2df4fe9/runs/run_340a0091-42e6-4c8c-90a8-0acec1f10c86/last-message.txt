The implementation I’d make is:
- Move the folder discovery logic out of the sessions route into a shared server helper.
- Return `folderOptions` from the projects page load.
- For each directory field, add:
  - a “Pick a known folder” select
  - the existing text input with `list=...` so users can still paste/type
  - selection wiring so picking an option fills the input
- Keep repo URL as plain text, but give `defaultRepoPath` the same picker treatment.

I would not use a browser-native `<input type="file" webkitdirectory">` here. In a web app it does not cleanly give you stable absolute server paths, which is what this form is storing. A server-backed directory picker/select is the correct model for this app.

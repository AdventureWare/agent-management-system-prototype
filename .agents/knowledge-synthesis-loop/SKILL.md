# Knowledge Synthesis Loop

## When to use this skill

- Use this skill after a burst of work, research, or experimentation.
- Use this skill when the user has too many notes, threads, or artifacts and needs structure.
- Use this skill when the goal is not just summary, but better future reuse.

## Workflow

1. Gather the evidence.
   List the source notes, threads, docs, artifacts, or repos that matter.
2. Separate facts from interpretation.
   Preserve what was observed, what was decided, and what is still inference.
3. Cluster the material.
   Group by recurring themes such as product insight, technical pattern, customer signal, operating lesson, or open risk.
4. Extract durable outputs.
   Prefer outputs such as:
   - principles
   - decision records
   - reusable checklists
   - candidate skills
   - follow-up tasks
5. Name what is still unresolved.
   Open questions are a first-class output, not a sign of failure.
6. End with the smallest useful knowledge update.
   Suggest which note, doc, skill, or task should actually be updated next.

## Output shape

- What happened
- Durable takeaways
- Reusable assets to create or update
- Open questions
- Immediate next actions

## Failure shields

- Do not stop at generic summary.
- Preserve provenance when the source matters.
- Do not collapse disagreements or uncertainty into false certainty.
- Prefer a few durable abstractions over a long transcript rewrite.

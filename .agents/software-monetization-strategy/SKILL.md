# Software Monetization Strategy

## When to use this skill

- Use this skill when the user needs to decide how a software product should make money.
- Use this skill when pricing, packaging, or monetization feels fuzzy, copied from another company, or disconnected from how the product actually creates value.
- Use this skill when the real question is model fit, not just what number to charge.

## Workflow

1. Clarify the product and buyer.
   Capture:
   - primary user
   - economic buyer
   - core recurring value
   - time to first value
   - current or expected acquisition path
   - major platform or billing constraints
2. Map the realistic model set.
   Compare only the models that plausibly fit the product:
   - one-time purchase
   - subscription
   - freemium
   - free trial
   - seat-based
   - usage-based
   - transaction-based
   - ad-supported
   - services-assisted
   - hybrid combinations
3. Judge fit on the few dimensions that matter most:
   - whether value is ongoing or front-loaded
   - whether price can scale with real customer value
   - cost-to-serve and gross-margin pressure
   - buyer predictability and budgeting comfort
   - self-serve versus sales-led motion
   - expansion path over time
4. Choose the value metric and package logic.
   State what should trigger payment, what scales spend, what remains free, and what creates the upgrade path.
5. Name the sharpest risks.
   Call out:
   - bill shock
   - weak willingness to pay
   - free-tier abuse
   - platform-policy constraints
   - bad value-metric fit
   - retention too weak for recurring revenue
6. End with a validation plan.
   Recommend the smallest useful test:
   - pricing interviews
   - paywall test
   - packaging comparison
   - usage-meter prototype
   - trial versus freemium experiment

## Output shape

- Current product and buyer reality
- Candidate model comparison
- Recommended monetization model and why
- Proposed value metric and packaging logic
- Main risks and rejected alternatives
- Validation plan and success metrics

## Failure shields

- Do not recommend a monetization model before stating what value is actually being sold.
- Do not confuse pricing model, packaging, and billing mechanics.
- Do not suggest recurring revenue when retention or ongoing value is still unproven.
- If app-store, payment, or marketplace rules matter, check current official documentation instead of relying on memory.

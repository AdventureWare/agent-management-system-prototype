# Worked Examples And Checks

Build examples and checks that make the learner think, not just watch.

This skill is for the teaching move after the topic is chosen: demonstration, contrast, guided practice, retrieval, and feedback.

## Workflow

1. Define the exact learning move.
   Decide whether the learner needs first exposure, error correction, transfer, memorization, or a confidence check.
2. Choose the minimal good example.
   Use the smallest example that still shows the real structure of the idea. Avoid decorative details that make the example memorable but misleading.
3. Pair example types intentionally.
   Combine at least two of these when useful:
   - worked example
   - partially worked example
   - near miss or non-example
   - comparison of correct vs incorrect reasoning
   - concrete case linked to the abstract rule
4. Add a self-explanation prompt.
   Ask the learner why a step was chosen, what rule applies, what would change in a variant, or how they would check the result.
5. Fade support.
   Move from fully worked -> partly worked -> independent attempt. Remove one support at a time.
6. Use short checks.
   Prefer one focused recall or transfer question over a large mixed quiz. Make the check diagnose a specific misunderstanding.
7. Give corrective feedback for the next attempt.
   Show what to change next, not just whether the answer is wrong.

## Example patterns

- `show structure`: a fully worked example with step labels
- `spot the break`: a near-correct example that fails at one important point
- `compare and choose`: two candidate answers with one better justified
- `change one variable`: hold most of the problem fixed and vary one element
- `explain the rule`: ask the learner to state the general idea behind the example
- `reverse direction`: give the answer and ask for the reasoning or setup

## Check design rules

- Match the check to the learning target, not to trivia around it.
- Keep the prompt short enough that failure means misunderstanding the idea, not misunderstanding the wording.
- Include at least one transfer prompt when the learner seems fluent on the original example.
- When using multiple choice, make distractors correspond to realistic misconceptions.
- When using analogy, state where it stops being reliable.

## Output shapes

- worked example
- partial worked example
- misconception probe
- mini quiz
- answer key with feedback
- example ladder from easy to transfer

## Failure shields

- Do not create examples that are easier than the real concept in a way that changes the concept.
- Do not test only surface recall when the learner needs transfer or reasoning.
- Do not use trick questions as understanding checks.
- Do not leave the learner with a correct answer but no reason why it is correct.
- Do not keep scaffolds in place so long that the learner never attempts independent performance.

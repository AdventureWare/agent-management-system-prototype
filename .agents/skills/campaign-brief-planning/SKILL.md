# campaign-brief-planning

## When to use this skill

- Use this skill when a campaign, launch, or promotion needs an execution-ready brief instead of loose ideas.
- Use this skill when strategy is mostly chosen and the task is to turn it into a scoped plan across surfaces and handoffs.
- Do not use this skill when the core audience, positioning, or offer is still unresolved. In that case, use strategy work first.

## Workflow

1. Confirm the fixed inputs: objective, audience, offer, promise, and the primary conversion event.
2. Write the campaign in one sentence before expanding the plan. If that sentence is fuzzy, the brief is not ready.
3. Map channels, asset types, landing or conversion surfaces, CTA flow, and any required supporting work.
4. Define what must be created, what can be reused, and what dependencies or approvals can block launch.
5. End with success metrics, review checkpoints, and a short now-next-later execution sequence.

## Project notes

- Keep one primary call to action per brief unless there is a strong reason to split intent.
- Make the landing surface and conversion path explicit; avoid campaign plans that stop at top-of-funnel activity.
- Prefer a small coordinated package that can ship and be measured over broad multi-channel sprawl.

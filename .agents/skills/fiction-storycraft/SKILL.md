# fiction-storycraft

## When to use this skill

- Use this skill when the task is to create, outline, or improve fictional narrative.
- Use this skill when a story needs stronger character desire, scene movement, subtext, voice, point of view, pacing, or endings.
- Use this skill when AI-generated fiction feels generic, inert, over-explained, or structurally formulaic.
- Do not use this skill for reported nonfiction, factual case studies, or evidence-bound narrative. Use a nonfiction skill for those.

## Workflow

1. Start with the story engine: who wants what, what resists them, and why it matters now.
2. Choose the telling stance deliberately: narrator, point of view, distance, and voice should fit the material rather than defaulting to generic competent prose.
3. Build scenes around pressure and change. Each important scene should have an objective, resistance, and a changed condition by the end.
4. Use subtext, dialogue, imagery, and motif to carry meaning indirectly instead of explaining everything at the surface.
5. Check structure and temporal design. Use a classic arc only if it fits; otherwise choose a pattern that matches the material while still preserving momentum.
6. Revise for force: cut explanatory slack, improve pacing, sharpen openings and endings, and make theme emerge from consequence rather than from messages.

## Project notes

- Prefer active desire over biography-heavy setup.
- Prefer concrete scene pressure over abstract lore or atmosphere.
- Treat theme as something dramatized through pattern and consequence, not pasted on as a takeaway.
- If the draft sounds like it could have been written by anyone, the likely problem is weak POV, weak voice, or weak selection of detail.

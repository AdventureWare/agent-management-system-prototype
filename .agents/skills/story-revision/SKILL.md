# story-revision

## When to use this skill

- Use this skill when a narrative draft exists but is not yet compelling.
- Use this skill when the writing feels generic, shapeless, over-explained, slow, tonally flat, or emotionally thin.
- Use this skill for both fictional and nonfictional narrative revision, as long as factual constraints are respected where relevant.
- Do not use this skill when the user first needs new reporting, research, or source validation instead of draft diagnosis.

## Workflow

1. Name the intended effect before editing: what should the reader feel, understand, or keep caring about?
2. Diagnose the draft by the highest-leverage failure, not by line-level nitpicking first. Common categories:
   - narrator / point of view / voice
   - character desire / scene pressure
   - dialogue / subtext
   - pacing / compression / temporal order
   - theme / imagery / ending
3. Fix structural and scene-level problems before polishing sentences.
4. Remove genericness by making perception, diction, and detail more specific to this narrator, subject, and situation.
5. Tighten explanation. Prefer implication, consequence, and image where the draft is saying too much too directly.
6. Re-check the opening and ending after revision, because they often become wrong once the middle improves.

## Project notes

- Treat revision as reconstruction, not cleanup.
- Prefer a short list of decisive changes over a long list of small edits.
- If the draft has no real scene pressure, better wording will not save it.
- If the draft sounds “AI,” the likely causes are weak stance, weak selection, weak compression, or weak scene logic.

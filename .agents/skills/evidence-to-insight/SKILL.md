# Evidence To Insight

## When to use this skill

- Use this skill after evidence has been gathered and the user needs a defensible synthesis.
- Use this skill when the user wants to communicate a point of view without drifting into unsupported claims.
- Use this skill when a memo, post, briefing, or recommendation should be grounded in legitimate information and data.

## Workflow

1. Inventory the evidence.
   List the strongest facts, data points, patterns, and disagreements first.
2. Separate claim types.
   Mark what is:
   - directly supported
   - inferred from multiple signals
   - plausible but still speculative
3. Ask what is actually new or useful.
   Insights should clarify implications, connect evidence, or expose tension. They should not just restate the notes.
4. Write claims proportionally.
   Strong evidence can support strong wording. Weak evidence requires narrower wording and explicit caveats.
5. Tie insight to action or contribution.
   End with what the user can now say, decide, test, teach, or build from the evidence.

## Output shape

- evidence summary
- core insights
- confidence and caveats
- implications or recommendations
- optional talking points or draft paragraphs

## Failure shields

- Do not launder opinion as evidence-backed insight.
- Do not present correlation as causation unless the evidence supports it.
- Do not bury disagreement, uncertainty, or weak support.
- If the evidence base is too thin for a strong conclusion, the honest output is a narrower claim or a request for more research.

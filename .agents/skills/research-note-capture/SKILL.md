# Research Note Capture

## When to use this skill

- Use this skill when the user wants research turned into notes they can reuse later.
- Use this skill when the output should preserve provenance rather than only a polished summary.
- Use this skill when the note format may be Obsidian markdown, plain markdown, or another structured knowledge artifact.
- Use this skill when learning value matters as much as the immediate answer.

## Workflow

1. Decide the note type.
   Choose the smallest useful format: source note, topic note, comparison note, research memo, or study brief.
2. Capture the source context first.
   Record title, author or organization, date, source type, link or file, and why it matters.
3. Separate note layers.
   Keep these distinct:
   - what the source says
   - what evidence or data it provides
   - your interpretation
   - open questions or follow-up actions
4. Preserve reusable specifics.
   Keep the numbers, definitions, frameworks, and contradictions that matter later.
5. Make the note easy to revisit.
   End with a short takeaway and the next thing worth reading, testing, or connecting.

## Obsidian guidance

- When the target is Obsidian, prefer plain markdown that stays portable.
- Use simple frontmatter only when it helps retrieval, such as `title`, `created`, `source_type`, `confidence`, and `tags`.
- Prefer stable headings like `Summary`, `Key Evidence`, `Interpretation`, `Open Questions`, and `Next Actions`.
- Use wikilinks only when the related note likely exists or should exist.

## Output shape

- source metadata
- concise summary
- key evidence or facts
- interpretation
- open questions
- next actions or related notes

## Failure shields

- Do not blur source facts and your own interpretation.
- Do not strip out provenance to make the note look cleaner.
- Do not over-format the note if a plain, portable structure will do.
- Do not keep everything; select for future usefulness.

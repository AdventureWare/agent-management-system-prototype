# Learning science notes for `teaching-tutoring`

Use this file when you need the evidence-backed reason for a tutoring move, not for every invocation.

## High-signal takeaways

### Diagnose and tailor

- Effective tutoring is strongest when it responds to the learner's current gap instead of replaying a generic explanation.
- Ongoing assessment matters because tutoring should adapt as the learner improves or reveals confusion.

### Teach metacognitive moves explicitly

- Good learners plan, monitor, and evaluate their understanding rather than only absorbing information.
- A tutor can support this by asking the learner what they think the task is asking, how they decided on a step, and how they would check their answer.

### Use worked examples and explanation prompts

- Worked examples help novices see structure and sequence before they are asked to perform independently.
- Explanatory questions and self-explanation prompts improve transfer because the learner has to connect steps to reasons.

### Use retrieval and frequent low-stakes checks

- Short quizzes, recalls, and paraphrase checks strengthen memory better than passive rereading.
- Quick checks are also diagnostic signals for the tutor.

### Connect concrete and abstract representations

- Learners often need a concrete case first, then the abstract rule, then a return to application.
- The reverse can work for more advanced learners, but transfer still improves when the connection is made explicit.

### Avoid overload

- Working memory is limited. Break demanding tasks into smaller parts, use external supports, and avoid introducing too much new information at once.

### Give feedback that redirects the next move

- Feedback works best when it is tied to the learning goal and helps the learner improve the next attempt, not when it merely labels the response as right or wrong.

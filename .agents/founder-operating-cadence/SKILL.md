# Founder Operating Cadence

## When to use this skill

- Use this skill for a weekly or periodic founder review.
- Use this skill when too many projects, ideas, or tasks are active at once.
- Use this skill when the user needs a clear continue, pause, kill, or escalate decision across current bets.

## Workflow

1. List the active bets.
   Capture the main product work, supporting efforts, side projects, challenge submissions, and important open opportunities.
2. State the current truth for each bet.
   For each item, write:
   - current objective
   - current status
   - last meaningful progress
   - main blocker or risk
   - next proof point
3. Check WIP pressure.
   Count how many meaningful efforts are active at the same time and identify where attention is fragmented.
4. Protect the main product.
   Explicitly ask whether each side effort strengthens, funds, validates, or distracts from the main product.
5. Force a decision on every active item.
   Use one of:
   - continue
   - pause
   - kill
   - escalate
6. Build the next-action runway.
   End with a short list of concrete next actions for the next operating window, not a broad backlog.

## Output shape

- Current bets table or list
- WIP and attention analysis
- Continue, pause, kill, or escalate decisions
- Main-product protection callouts
- Next-action runway

## Failure shields

- Do not let every item remain "active."
- If a bet has no next proof point, it is probably not ready to stay in active WIP.
- Do not keep low-leverage side work alive by inertia.
- Prefer a small number of clearly advancing bets over broad optionality theater.
